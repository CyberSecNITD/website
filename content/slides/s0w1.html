To a Reverse Engineer

To a Developer

  • lines of code in programming language of their choice
  • Stack, Strings, For-loop, Linked List, Classes and so on..

To a End User

  • To Users, its this huge “.exe” file that you double click and things run magically.

To a Reverse Engineer

  • To a Reverse Engineer, its both.
    • They work on the “.exe” but understand or develop and idea of how it must have been coded.
    • For example, if you want to hack a game, get infinite health, probably you ll find the function or DS responsible for storing the health value which is an integer ( as a developer ) stored as binary code ( as a user ). Then rewrite or patch the “.exe” to let’s say keep health to MAX_INT value.
